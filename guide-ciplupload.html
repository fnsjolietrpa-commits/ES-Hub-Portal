📦 WMS CIPL Uploader

CI/PL(PDF·Excel) 파일을 PO 번호 기준으로 WMS에 자동 업로드

💾 다운로드 (SharePoint) WMS_CIPL_Uploader.zip · 약 69.5MB

⬇️ 설치 가이드(소요 시간 2분)

1

위 다운로드 버튼 클릭

위쪽 파란색 다운로드 버튼을 누르면 회사 SharePoint 페이지로 이동합니다.

2

회사 계정으로 로그인

FNS 회사 계정(이메일)으로 로그인하세요. 이미 로그인되어 있으면 이 단계는 건너뜁니다.

3

다운로드 진행

SharePoint에서 좌측 상단 다운로드 버튼을 클릭하여 다운로드합니다.

4

압축 해제

다운로드된 WMS_CIPL_Uploader.zip 파일을 우클릭 → "압축 풀기"후 WMS_CIPL_Uploader.exe을 실행합니다.

5

처음 실행 시 Windows 디펜더 경고

EXE를 더블클릭하면 처음 한 번은 아래와 같은 경고가 뜹니다. 이건 정상이며, 코드 서명 인증서가 없는 EXE에 항상 나타납니다.

"More info"를 클릭하면 "Run anyway" 버튼이 나타납니다. 이걸 누르면 실행됩니다.

💡 한 번만 하면 끝: 처음 실행 후엔 이 경고가 다시 뜨지 않습니다.
6

그래도 실행이 막힌다면 (차단 해제)

Windows 스마트 스크린/스마트 앱 컨트롤이 더 강하게 막아서 "Run anyway" 버튼이 안 보이거나 실행이 계속 차단될 때가 있습니다. 이럴 땐 파일 자체의 차단을 풀어주면 됩니다.

WMS_CIPL_Uploader.exe 우클릭 → "속성(Properties)" → 하단 "차단 해제(Unblock)" 체크 → "확인" 후 다시 실행하세요.

"차단 해제" 항목이 안 보이면 이미 차단이 풀린 상태이거나 처음 경고(5번) 단계에서 해결된 것이므로, 5번의 More info → Run anyway로 실행하면 됩니다.

🚀 사용 가이드

EXE를 실행하면 아래와 같은 창이 열립니다. 폴더 안의 CI/PL 파일을 파일명의 PO 번호 기준으로 WMS(wms.itsscm.com)에 자동 업로드합니다.

파일명에 PO가 없다면, 사내 포털내의 "선적서류 파일명 컨버터"를 먼저 사용하여 파일명을 PO로 변환하시기 바랍니다.
1

WMS 아이디 / 비밀번호 입력

Username · Password에는 본인의 WMS(wms.itsscm.com) 로그인 아이디와 비밀번호를 입력합니다. (사내 WMS 계정과 동일)

💡 Save ID/PW: 체크해두면 아이디·비밀번호가 PC에 저장되어 다음 실행 때 자동으로 채워집니다.
2

대상 폴더 선택

"Select Folder" 옆의 📁 버튼을 클릭해서 업로드할 CI/PL 파일들이 들어 있는 폴더를 선택합니다.

  • 자동 인식 파일 — .pdf · .xls · .xlsx
  • 하위 폴더 포함 — 선택한 폴더 안의 하위 폴더까지 자동으로 검색합니다
파일이 들어있는 "폴더"를 선택하는 것이므로 폴더 안 파일이 보이지 않더라도 정상 동작입니다.
3

PO Number 입력 (선택사항)

특정 PO만 올리고 싶을 때 사용합니다. PO 번호를 쉼표(,) 또는 줄바꿈으로 구분해서 여러 개 입력할 수 있습니다.

  • 비워두면 — 폴더 안 모든 파일의 PO를 자동 인식해서 전부 업로드
  • 입력하면 — 입력한 PO에 해당하는 파일만 업로드

파일명에서 PO 번호를 자동으로 추출합니다. 인식하는 형식은 다음과 같습니다:

  • FNS PO — FNSPWS + 영문 1자 + 숫자 (예: FNSPWSI26031600004)
  • MI PO — 457로 시작하는 10자리 숫자 (예: 4570249952)
4

Run Script 클릭

"Run Script" 버튼을 누르면 아래 순서로 자동 진행됩니다. 진행 상황은 우측 Console Output과 하단 진행률 바에 실시간으로 표시됩니다.

  • 1단계 · 파일 매핑 — 파일명에서 PO를 추출해 파일과 연결합니다. (PO를 못 찾은 파일은 콘솔에 [PO 없음]으로 표시되며 업로드되지 않습니다)
  • 2단계 · WMS 업로드 — 크롬이 자동 실행되어 WMS 로그인 → 창고 Chicago HUB WH 설정 → Outbound > Outbound CC Request 이동 → PO별로 검색 후 파일을 업로드합니다.
작업 중에는 크롬 창이 화면 밖에 숨겨진 채 백그라운드로 진행됩니다. 창이 보이지 않아도 정상입니다.
5

결과 확인 팝업

업로드가 끝나면 "WMS 업로드 결과를 확인하시겠습니까?" 팝업이 뜹니다.

  • 예 숨겨졌던 크롬 창이 앞으로 나오고, 방금 업로드한 PO들을 WMS에서 자동으로 검색해 결과를 보여줍니다.
  • 아니오 크롬 브라우저를 종료합니다.
6

Console Output 읽는 법

우측 콘솔에 모든 진행 상황과 오류가 색상으로 표시됩니다.

  • [OK] 완료 · 성공 (초록)
  • [INFO] 진행 · 단계 · 매핑 (파랑)
  • [ERROR] 실패 · 오류 (빨강)

💡 알아두면 편한 동작

  • 실패 PO 자동 채움 — 업로드에 실패한 PO는 PO Number 입력칸에 자동으로 다시 채워집니다. 그대로 Run Script를 다시 누르면 실패한 PO만 재시도합니다.
  • 중복 업로드 방지 — 같은 세션에서 이미 올린 파일은 자동으로 건너뜁니다.
  • 브라우저 재사용 — 한 번 로그인한 크롬 세션은 재사용되어, 다시 실행할 때 로그인·메뉴 이동 단계를 건너뜁니다.
  • 이미 다 올라간 경우 — 모든 파일이 이미 업로드돼 있으면 바로 결과 확인 팝업만 띄웁니다.
