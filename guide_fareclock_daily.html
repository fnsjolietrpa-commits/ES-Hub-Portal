📖 어떤 일을 하나요?

▼
1

FareClock 자동 로그인 — 전일 데이터 기준

컴퓨터가 알아서 FareClock에 접속하고, 날짜를 어제로 바꿔 출퇴근 기록을 불러옵니다.

⚠️ 어제 데이터가 없는 경우(예: 공휴일): 모든 담당자에게 "데이터 없음" 안내 이메일을 발송하고 종료합니다.
2

부서(Shift Class)별 파일 분리

전체 데이터를 부서 단위로 분리해 각 부서의 출퇴근 기록만 따로 정리합니다. 처음 보는 새 부서가 생기면 관리자에게 별도로 알림 이메일을 보냅니다.

3

팀별 담당자 개별 이메일 발송

각 부서 담당자에게는 해당 팀 데이터만 담긴 파일을 첨부해서 이메일을 발송합니다. 근무 시간이 기준에서 벗어난 직원은 색으로 표시됩니다.

  • 노란색 — 근무 시간이 7시간 40분 이하 (짧음)
  • 빨간색 — 근무 시간이 8시간 15분 이상 (초과)
4

전체 통합 리포트 발송

모든 부서 데이터가 담긴 통합 파일을 관리자 전체 배포 목록에 발송합니다. 이메일에는 부서별 FL / GL 인원 수, 정규 근무시간, 초과 근무시간 요약표가 포함됩니다.

💡 하루 한 번만 실행: 같은 날 두 번 실행돼도 이메일이 중복 발송되지 않습니다.

English Guide

▼

📖 What it does

▼
1

FareClock Login — Previous Day

The computer signs into FareClock on its own and switches the date to yesterday to pull the previous day's time cards.

⚠️ No data for yesterday (e.g. a holiday): A "No Data" notification is emailed to all recipients and the script stops.
2

Data Split by Shift Class

Separates the full dataset by Shift Class so each team's records are kept on their own. If a brand-new Shift Class appears, the admin gets an alert email automatically.

3

Individual Email per Shift Class

Each Shift Class manager receives only their team's data as an email attachment. Workers with unusual hours are highlighted in color.

  • Yellow — worked 7 hrs 40 min or less (short hours)
  • Red — worked 8 hrs 15 min or more (overtime)
4

Full Combined Report to All Managers

A single Excel file with all teams in separate tabs is emailed to the full management list, with FL/GL headcount, regular hours, and overtime hours per team.

💡 Runs only once a day: Even if triggered twice, the emails only go out once.
