📋 결과 파일(3_Result) 읽는 법

▼

결과는 재실행 중의 자체 판단이 아니라, 실행이 끝난 뒤 NERP를 다시 조회해서 확인한 실제 상태를 기준으로 기록됩니다.

success

이번 실행에서 정상적으로 삭제 확인됨.

already done

이번 실행이 하기 전에 이미 처리되어 있던 팔렛 (이번 실행이 삭제한 게 아님).

failed

재확인했지만 아직 삭제되지 않은 것으로 확인됨 — NERP에서 직접 확인 필요.

unverified

재확인 조회 자체가 실패함 (성공/실패 여부 불명) — NERP에서 직접 확인 필요.

possible error

요청하지 않은 팔렛인데 이번 실행 중 처리된 것으로 확인됨 — 빨간 글씨로 맨 위에 표시. 가장 심각한 케이스이니 즉시 확인 필요.

⬇️ Installation (first time only)

▼
1

Click the download button above

It takes you to the company SharePoint page to download NERP_Unboxing_LIVE_DELETE.zip.

2

Unzip into a permanent working folder

Whatever folder you run this exe from becomes its "working folder" — the request, review, and result files all accumulate there. Extract it into a fixed location (e.g. a shared drive) and always run it from that same folder.

⚠️ Important: The _internal and ms-playwright folders must always stay together with NERP_Unboxing_LIVE_DELETE.exe. Copying the .exe by itself will not work.
3

Windows security warning (first run)

A blue SmartScreen warning may appear the first time. This is normal for unsigned EXE files. Click "More info" → "Run anyway".

💡 One time only: This warning will not appear again after the first run.
4

If still blocked — unblock the file

Right-click NERP_Unboxing_LIVE_DELETE.exe → Properties → check "Unblock" at the bottom → OK, then run again.

5

Log in to NERP

Running the exe opens a login popup directly (no console window). Enter your NERP ID and password.

NERP Unboxing Login

ID: [________________] Password: [________________] ☑ Save ID

Cancel
Login
💡 Save ID pre-fills your ID next time. Your password is never saved — you'll always need to type it in.

📖 How it works — the 3-step flow

▼

One run of the exe chains Request → Review → Execute automatically. Closing the window pauses the session — running the exe again will offer to resume from where you left off.

1

Step 1 — Request

After logging in to NERP, it downloads the full inventory.

In the opened Excel file, enter the number of pallets to unbox per material in the yellow "Requested Pallets" column, save, and close Excel.

No.MaterialAvailable PalletsRequested Pallets
1MPD00268AB425
2MHU00844AB183
3MPD00250AB27
Only enter numbers in the yellow cells — leave a material blank if you don't need it.

Click Continue in the program window.

Pallets already processed (Repack Status done) are automatically excluded from the request/review.
2

Step 2 — Review

2_Review_YYYYMMDD_HHMM.xlsx opens automatically, showing recommended/excluded pallets per material.

Check the selected pallets — moving the O to a different row selects that pallet instead.

SelectedVerdictHU External ID
ORecommended...26311001
OSwappable...26311003
Recommended...26311002
Move the O mark to another row to select that pallet instead.

Save, close Excel, and click Continue in the program window. (If you need to change the pallet counts, click Edit pallet count to go back to Step 2.)

3

Step 3 — Execute ⚠️ Real deletion

A final confirmation popup shows the pallet count/quantity per material.

Confirm

MPD00268AB: 5 pallets, 840 EA Total quantity: 840 EA Proceed with execution? (y/N):

Yes
No

Clicking Yes logs back into NERP and runs the actual Delete Box Data action per material.

Once execution finishes, the results are shown in an Excel file.

⚠️ Once you click Yes, it cannot be undone. Data is permanently deleted in NERP.

📋 Reading the result file (3_Result)

▼

Results are based on re-querying NERP after execution, not on any in-process signal from the delete loop itself.

success

Confirmed deleted by this run.

already done

Was already done before this run reached it (not deleted by this run).

failed

Re-checked and confirmed NOT yet deleted — check NERP directly.

unverified

The re-check query itself failed (success/failure unknown) — check NERP directly.

possible error

A pallet that was NOT requested was processed during this run — shown in red at the top. The most serious case; check immediately.

🔁 Pausing / Resuming · Common Issues

▼
1

You can stop at any time

Closing the log window pauses right there. Running the exe again in the same folder shows "[Found] ... Continue / Start from Step 1".

💡 Pallets already marked success/already done are never re-attempted on resume.
2

"File still open in Excel" popup

Shown when a generated file was closed without saving, or is still open. Save and close it, then click Retry.

3

NERP responding slowly

Slower-than-usual NERP queries or screen transitions are retried automatically. If it keeps failing, check your NERP connection/network.

Questions or issues? Contact your administrator.
